# Mark Data Reception

## Purpose

接收来自车机的标记数据（时间戳 + GPS + 道路名称），支持扫码接收和模拟数据两种方式，为视频裁剪提供原始数据。

## Scenario: 扫码接收标记数据

**GIVEN** 用户在首页点击"扫码接收"
**WHEN** 用户扫描包含有效 JSON 格式标记数据的二维码
**THEN** 系统解析二维码数据并将标记列表存入 globalData.marks，页面显示已接收标记点数量，用户可点击"开始裁剪视频"进入裁剪页

## Scenario: 扫码结果为空

**GIVEN** 用户在首页点击"扫码接收"
**WHEN** 用户扫码但二维码内容为空
**THEN** 系统提示"扫码结果为空"，不进行页面跳转

## Scenario: 扫码数据解析失败

**GIVEN** 用户在首页点击"扫码接收"
**WHEN** 用户扫描的二维码不是有效 JSON 格式
**THEN** 系统提示"数据解析失败"，不更新标记数据

## Scenario: 使用模拟数据测试

**GIVEN** 用户在首页（未连接车机）
**WHEN** 用户点击"使用模拟数据"
**THEN** 系统加载 mock-data.js 中的 2 个标记点，页面显示"已接收 2 个标记点"，用户可进入裁剪流程

## Scenario: 无标记数据时禁止进入裁剪

**GIVEN** 用户未接收任何标记数据
**WHEN** 用户直接点击"开始裁剪视频"
**THEN** 系统提示"请先扫码或加载模拟数据"，不进行页面跳转
