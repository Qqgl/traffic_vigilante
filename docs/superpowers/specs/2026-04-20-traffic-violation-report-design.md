# 交通违法举报系统设计

## 1. 项目概述

**项目名称：** 交通违法举报（traffic_vigilante）

**核心功能：** 通过行车记录仪视频举报交通违法行为。用户开车时一键打点标记可疑行为，到达目的地后通过小程序裁剪视频片段并提交12123举报。

**目标平台：**
- 比亚迪车机 App（DiLink 平台）
- 微信小程序

---

## 2. 整体架构

```
┌─────────────────────┐      二维码（标记数据）      ┌─────────────────────┐
│   比亚迪车机 App      │ ───────────────────────→  │   微信小程序         │
│                      │                            │                     │
│  · 监听打点按键        │   时间戳列表               │  · 扫码接收标记      │
│  · 记录时间戳          │   +视频文件路径            │  · 双视频时间线裁剪  │
│  · 匹配行车记录视频    │                            │  · 选择违法类型      │
│  · 生成二维码          │                            │  · 生成12123举报包  │
│                      │                            │  · 跳转12123 App    │
└─────────────────────┘                            └─────────────────────┘
```

## 3. 车机 App 设计

### 3.1 核心功能

| 功能 | 说明 |
|------|------|
| 打点记录 | 行车中一键打点，记录时间戳（利用原生行车记录仪，不额外录像） |
| 位置记录 | 打点时同时记录GPS坐标，并通过高德地图逆地理编码获取当前道路名称 |
| 标记列表 | 本次行程所有标记点（时间戳 + 位置 + 道路名称） |
| 自动匹配 | 到达目的地后，自动将时间戳匹配到对应的5分钟视频文件 |
| 生成二维码 | 导出标记数据（时间戳 + 位置 + 道路名称 + 对应视频文件信息） |

### 3.2 数据模型

**标记点：**
```json
{
  "timestamp": "2026-04-20T10:30:05",
  "gps": { "lat": 31.2304, "lng": 121.4737 },
  "roadName": "京沪高速上海方向",
  "videoFile": "/path/to/record_20260420_103000_103500.mp4",
  "offsetInFile": 5
}
```

**行程标记数据（导出为二维码）：**
```json
{
  "tripId": "uuid",
  "startTime": "2026-04-20T09:00:00",
  "endTime": "2026-04-20T10:45:00",
  "marks": [
    { "timestamp": "2026-04-20T10:30:05", "gps": { "lat": 31.2304, "lng": 121.4737 }, "roadName": "京沪高速上海方向", "videoFile": "record_xxx.mp4", "offset": 5 }
  ]
}
```

### 3.3 界面交互

- **开车时：** 悬浮小球或极简界面，打点按钮醒目
- **打点时：** 震动/声音反馈，记录时间戳 + GPS坐标 + 当前道路名称（由高德地图逆地理编码获取）
- **行程结束：** 自动列出标记片段预览（显示时间 + 道路名称），生成二维码

## 4. 微信小程序设计

### 4.1 核心功能

| 功能 | 说明 |
|------|------|
| 扫码接收 | 扫描车机生成的二维码，接收标记数据（时间戳 + 位置 + 道路名称） |
| 视频获取 | 通过车机热点WiFi拉取视频，或U盘导入 |
| 路段确认 | 显示自动获取的道路名称供用户确认/修改 |
| 双视频时间线 | 同时播放两个视频文件，用户拖拽选择起止点 |
| 违法类型选择 | 列表选择（非法变道/压线/闯红灯/违法停车/应急车道行驶） |
| 生成举报包 | 符合12123格式的视频片段 + 信息文件 |
| 跳转12123 | 唤起12123 App进行上传 |

### 4.2 违法类型

- 非法变道
- 压线行驶
- 闯红灯
- 违法停车
- 应急车道行驶

### 4.3 视频裁剪逻辑

- 起始时间 = 标记时间 - 用户可调的缓冲时间（默认3秒）
- 结束时间 = 用户手动拖拽时间线决定
- 若跨两个5分钟视频文件，自动拼接

### 4.4 举报包格式

生成符合12123要求的文件包（视频 + JSON信息文件），用户一键跳转上传。

## 5. 技术选型

| 模块 | 技术 |
|------|------|
| 车机 App | 比亚迪 DiLink 原生开发 |
| 小程序 | 微信原生框架 |
| 数据传输 | 二维码（标记数据）/ 车机热点WiFi（视频） |

---

## 6. 实现优先级

**第一阶段（小程序先行）：**
1. 微信小程序开发（扫码接收 → 时间线裁剪 → 生成举报包 → 跳转12123）
2. 使用模拟数据测试举报流程

**第二阶段（车机端）：**
1. 比亚迪车机 App 开发（打点记录 + 自动匹配 + 生成二维码）
2. 与小程序联调
