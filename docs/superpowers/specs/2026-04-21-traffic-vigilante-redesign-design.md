# 交通违法举报小程序 - 整改设计

## 背景

12123 不支持直接举报违法行为，各地都有自己的举报平台（如成都的蓉e行）。用户需要在地方平台上手动填表上传视频。

**核心价值重新定位：** 小程序是辅助剪辑工具，帮助用户快速找到并导出违法视频片段。

## 业务流程

```
行车记录仪标记违法
    ↓
生成二维码（含标记点数据）+ 视频文件
    ↓
手机连行车记录仪 WiFi → 扫码接收标记数据
    ↓
在微信内直接访问行车记录仪上的视频
    ↓
辅助剪辑：标记时间点 → 预览 → 调整出入点 → 导出视频
    ↓
用户到地方平台（蓉e行）手动填表上传
```

## 页面设计

### 1. 首页 (index)

- 扫码接收按钮
- 使用模拟数据按钮（调试用）
- 显示已接收标记点数量
- 进入裁剪视频按钮

### 2. 裁剪页 (clip)

- 标记点导航（上一个/下一个）
- 显示当前标记点信息（时间、路段）
- 视频预览（连行车记录仪 WiFi 时可直接播放）
- 裁剪参数控制：
  - 标记前缓冲时间
  - 裁剪开始/结束时间
- 确认裁剪按钮 → 跳转确认页

### 3. 确认页 (confirm) - 简化

- 确认道路名称
- 显示裁剪时间范围
- **移除违法类型选择**
- 确认导出按钮

### 4. 结果页 (result) - 重新设计

- 显示"视频片段已导出"
- 显示举报信息摘要（时间、地点）
- 跳转蓉e行按钮
- 返回首页按钮

### 5. 扫码页 (scan)

- 扫码接收标记数据
- 解析二维码 → 存储到 globalData
- 自动返回首页

## 数据流

```js
// 全局数据
globalData = {
  marks: [...],           // 标记点数组
  videoAccessUrl: '',     // 行车记录仪视频访问地址
  clipParams: {
    mark: {...},
    clipStart: number,
    clipEnd: number
  }
}
```

## 导出流程

1. 用户在确认页点击"确认导出"
2. 调用微信 `wx.saveVideoToPhotosAlbum` 保存视频到相册
3. 跳转到结果页，显示导出成功
4. 结果页提供蓉e行入口（后续可添加城市选择）
